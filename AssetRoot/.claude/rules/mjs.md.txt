# PlayCanvas mjs スクリプトのコーディング規約

## 命名規則

- ファイル名・`scriptName` は `[接頭辞]_camelCase`（例: `BH_camelCase`）。接頭辞: `BH_*` Entity挙動、`SH_*` シェーダー、`EF_*` エフェクト、`PP_*` ポストプロセス。
- 新規作成時は scriptName が `bHCamelCase` になるので、表記統一のため `BH_camelCase` に修正する。

```javascript
export class BHCamelCase extends Script {
    static scriptName = 'BH_camelCase';
}
```

## コメント

- JSDoc（`/** ... */`）を推奨。説明とタグ（`@private`、`@returns` 等）を含める。public の場合は必ず JSDoc をつける。private 関数の場合は文脈が複雑な場合（10行以上）に JSDoc を付ける。
- 関数内には原則コメントをつけない。
- `var` の使用を避ける。

## 属性定義

[PlayCanvas ESM リファレンス](https://developer.playcanvas.com/ja/user-manual/scripting/script-attributes/esm/)に従う。`static attributes` は使わず、クラスプロパティで定義する。

- **基本**: `@attribute` でEditorに公開。初期値がなければ `@type` 必須（ないと無視される）。JSDoc先頭の1文がツールチップ、`@title` で表示名。
- **型**: `number`・`string`・`boolean`・`Vec2`～`Vec4`・`Color`・`Curve`・`Asset`・`Entity`。`Vec*`／`Color`／`Curve`／`Asset`／`Entity` は `playcanvas` から import。
- **タグ**: 数値は `@range [min, max]` 等。Asset は `@resource texture` 等。文字列は `@placeholder`。条件表示は `@enabledif`／`@visibleif`。配列は `@type {T[]}` と `@size`。ゲッター/セッターはペアの直前に `@attribute` を1回。

```javascript
/**
 * 属性の説明
 * @attribute
 * @type {pc.Entity}
 */
propertyName = defaultValue;
```

## フォールバック禁止（対象: 重要オブジェクトの解決）

- player, camera 等の**動作に不可欠なオブジェクトが取得できない場合、別の名前・別のパスで探しに行くフォールバックを使わない**。バグを隠す原因になる。
- 取得できなかった場合は早期リターンまたはエラーログで明示的に失敗させること。
- 以下は禁止対象外:
  - UI表示用のデフォルト値（`|| ''`, `|| 0`, `?? defaultValue`）
  - API互換のためのコンポーネント切り替え（`render || model`）
  - 遅延解決で `null` を返し、後続で明示的にチェックするパターン

```javascript
// NG: 別の名前で探しに行くフォールバック（失敗原因を隠す）
if (!cam) cam = this.app.root.findByName('MainCamera') || this.app.root.findByName('Camera');

// NG: 重要オブジェクトへのフォールバック
const pos = this._player?.getPosition() || this._camera?.getPosition() || new Vec3(0, 0, 0);

// OK: 明示的に失敗
if (!this._player) {
    logger.error('method', 'player not resolved');
    return;
}
const pos = this._player.getPosition();

// OK: UI表示用デフォルト値
const name = this.profile.displayName || 'Unknown';

// OK: API互換
const render = this.entity.render || this.entity.model;

// OK: 遅延解決 → null（後続でnullチェックする前提）
this._client = this.matchEntity?.script?.CORE_Match?.client || null;
```
