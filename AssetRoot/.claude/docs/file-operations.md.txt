# PlayCanvas ファイル操作リファレンス

## 基本方針
- **新規作成**: Editor API経由（createScript, createText, createFolder等）
- **編集**: ローカルファイルを直接編集（Edit/Write）→ ユーザーが手動でセーブ
- **削除**: Editor API経由
- **スクリプトパース**: 必要時に `/parse-scripts` コマンドで手動実行

## Editor API（ブラウザ経由の操作）

PlayCanvas Editorがブラウザで開いている状態で、playwright-cli経由で実行する。

### スクリプト作成
```bash
playwright-cli eval "window.editor.assets.createScript('BH_example.mjs'), 'done'"
```

### テキストファイル作成（.md等）
```bash
# folder引数はアセットオブジェクト（IDは不可）
playwright-cli eval "() => { var folder = window.editor.assets.get(FOLDER_ID); window.editor.assets.createText({ name: 'file.md', text: ' ', folder: folder }); return 'done'; }"
```
- `createText` で作られたアセットは内部的に `.txt` が付く（例: `MEMORY.md` → `MEMORY.md.txt`）

### フォルダ作成
```bash
playwright-cli eval "() => { var parent = window.editor.assets.get(PARENT_ID); window.editor.assets.createFolder({ name: 'name', folder: parent }); return 'done'; }"
```

### ファイル削除（配列で渡す）
```bash
playwright-cli eval "() => { var a = window.editor.assets.get(ASSET_ID); window.editor.assets.delete([a]); return 'done'; }"
```

### Preload設定
```bash
playwright-cli eval "() => { var a = window.editor.assets.get(ASSET_ID); a.set('preload', true); return 'done'; }"
```

### スクリプトパース
```bash
playwright-cli eval "() => { var a = window.editor.assets.get(ASSET_ID); return window.editor.assets.parseScriptCallback(a).then(function(r) { return JSON.stringify(r); }); }"
```
- パースが必要なタイミング: スクリプト新規作成時、attribute パラメータ変更時

### 長い内容をbase64で渡す（UTF-8対応デコード必須）
```bash
B64=$(cat file | base64 -w0)
playwright-cli eval "() => { var folder = window.editor.assets.get(FOLDER_ID); var b64 = '$B64'; var text = new TextDecoder().decode(Uint8Array.from(atob(b64), function(c) { return c.charCodeAt(0); })); window.editor.assets.createText({ name: 'file.md', text: text, folder: folder }); return 'done'; }"
```

## スクリプト作成ワークフロー

1. Editor APIでスクリプトアセットを作成
2. Preloadをtrueに設定
3. VSCode PlayCanvas拡張でローカルに同期されるのを待つ
4. ローカルファイルにESMスクリプトの内容を記述（Edit/Write）
5. ユーザーが手動でセーブ → パースも自動実行される

### ESMスクリプトの必須要素
- `import * as pc from 'playcanvas'` (pc.Vec3の様に参照)
- `export class ClassName extends Script { ... }`
- `static scriptName = 'scriptName'` が必須

## 動的情報の取得

```bash
# アセットIDを名前で検索
playwright-cli eval "() => { var a = window.editor.assets.list().find(function(a) { return a.get('name') === 'TARGET_NAME'; }); return a ? a.get('id') + '' : 'not found'; }"
```

### フォルダIDのキャッシュ
フォルダIDは AutoMemory にキャッシュし、毎回の playwright-cli 呼び出しを削減する。

**保存先**: AutoMemory (`MEMORY.md`)
**フォーマット**:

```markdown
# PlayCanvasEditor キャッシュ

## フォルダID
- .claude/: 123456
- .claude/commands/: 123457
- .claude/docs/: 123458
- .claude/rules/: 123459
- Scripts/: 123460
- Scripts/Behaviors/: 123461
```

**更新ルール**: 初回アクセス時に取得・蓄積。not found時に再取得。

## アセット階層構造

- **`parent` プロパティは常に `null`** — フォルダ階層の判定には使えない
- **`path` プロパティ（祖先フォルダIDの配列）** が実際の階層を表す
  - `path=[]` → ルート直下
  - `path=[A]` → フォルダAの子
  - `path=[A,B]` → フォルダBの子（BはAの中）
- 親フォルダIDの取得: `path[path.length - 1]` （`path`が空ならルート）
